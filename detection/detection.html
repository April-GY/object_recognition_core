Using the different trained objects, we can now detect them.

Use¶

Just run the detection.py script in /apps. This will run continuously on the input image/point cloud.

The server requires a configuration file through the -c option. Some of the options in there can be overriden by the command line for convenience. Use a config file from one of the pipelines or create your own and change the following parameters to your needs:

  • the object_ids should be the list of object ids you want to train on. If you want, you can also use object_names, that are more human readable

Just rosrun:

rosrun object_recognition_server server.py -c config_detection.sample

This will start a server with a given configuration file. If you want to test the server, just execute the client once:

rosrun object_recognition_server client.py

You can also use roslaunch if you want traditional actionlib support. There is a config_file argument that can help you choose different pipelines:

roslaunch object_recognition_server server.robot.launch

Command Line Interface¶

usage: detection [-h] [-c CONFIG_FILE] [--niter ITERATIONS] [--shell] [--gui]
                 [--logfile LOGFILE] [--graphviz] [--dotfile DOTFILE]
                 [--stats]

optional arguments:
  -h, --help            show this help message and exit
  -c CONFIG_FILE, --config_file CONFIG_FILE
                        Config file

Ecto runtime parameters:
  --niter ITERATIONS    Run the graph for niter iterations. 0 means run until
                        stopped by a cell or external forces. (default: 0)
  --shell               'Bring up an ipython prompt, and execute
                        asynchronously.(default: False)
  --gui                 Bring up a gui to help execute the plasm.
  --logfile LOGFILE     Log to the given file, use tail -f LOGFILE to see the
                        live output. May be useful in combination with --shell
  --graphviz            Show the graphviz of the plasm. (default: False)
  --dotfile DOTFILE     Output a graph in dot format to the given file. If no
                        file is given, no output will be generated. (default:
                        )
  --stats               Show the runtime statistics of the plasm.

Configuration File¶

The configuration file is where you define your graph and with the current ORK, you can choose any of the following sources:

set(['object_recognition_capture', 'object_recognition_core', 'object_recognition_msgs', 'object_recognition_tod', 'object_recognition_ros', 'object_recognition_linemod', 'object_recognition_transparent_objects', 'object_recognition_tabletop', 'object_recognition_by_parts'])
detector: declare_params
set([<class 'object_recognition_core.io.source.OpenNI'>, <class 'object_recognition_ros.io.source.ros_kinect.RosKinect'>, <class 'object_recognition_ros.io.source.bag_reader.BagReader'>])

any of the following sinks:

set(['object_recognition_capture', 'object_recognition_core', 'object_recognition_msgs', 'object_recognition_tod', 'object_recognition_ros', 'object_recognition_linemod', 'object_recognition_transparent_objects', 'object_recognition_tabletop', 'object_recognition_by_parts'])
detector: declare_params
set([<class 'object_recognition_core.io.sink.GuessCsvWriter'>, <class 'object_recognition_ros.io.sink.publisher.Publisher'>, <class 'object_recognition_tabletop.table_publisher.TablePublisher'>])

or the following pipelines:

set(['object_recognition_capture', 'object_recognition_core', 'object_recognition_msgs', 'object_recognition_tod', 'object_recognition_ros', 'object_recognition_linemod', 'object_recognition_transparent_objects', 'object_recognition_tabletop', 'object_recognition_by_parts'])
detector: declare_params
set([<class 'object_recognition_transparent_objects.detector.TransparentObjectsDetector'>, <class 'object_recognition_tabletop.detector.TabletopObjectDetector'>, <class 'object_recognition_by_parts.detector.PartsBasedDetector'>, <class 'object_recognition_linemod.detector.LinemodDetector'>, <class 'object_recognition_tabletop.detector.TabletopTableDetector'>, <class 'object_recognition_tod.detector.TodDetector'>])

More of any of those can be added by the user obviously
